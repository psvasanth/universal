# Universal — Daily Command

A dark, installable PWA: calendar, event countdowns, an Eisenhower task matrix, and settings — built with React, Tailwind CSS, and Vite.

## 1. Install and run locally

```bash
npm install
npm run dev
```

Open the printed `http://localhost:5173` URL. On your phone, connect to the same Wi-Fi and open your computer's local IP instead (e.g. `http://192.168.1.23:5173`) to test on Android directly — `devOptions.enabled` in `vite.config.js` makes the service worker active in dev too, so install/offline behavior is testable right away.

## 2. Build for production

```bash
npm run build
npm run preview -- --host
```

`npm run build` outputs a static site to `dist/` — this is what you deploy. `vite-plugin-pwa` automatically generates:
- `dist/manifest.webmanifest` — app name, icons, colors, display mode
- `dist/sw.js` — the service worker (offline caching, auto-update)
- registration code injected into `index.html`

## 3. Push this project to GitHub

This repo already includes `.github/workflows/deploy.yml` — a GitHub Actions workflow that runs `npm install` and `npm run build` with Node.js and publishes the result to GitHub Pages automatically, every time you push to `main`. You don't need to build or upload anything by hand.

### Easiest way: GitHub Desktop (no command line)

1. Install **GitHub Desktop** from desktop.github.com and sign in with your GitHub account.
2. In GitHub Desktop: **File → Add Local Repository** → browse to your `nova-pwa` folder → if it says "this isn't a git repository," click **create a repository** in that same dialog.
3. Click **Publish repository** (top bar). Name it (e.g. `nova-pwa`), choose Public or Private, and publish.
4. On GitHub.com, open your new repo → **Settings** → **Pages** (left sidebar) → under "Build and deployment," set **Source** to **GitHub Actions**.
5. Go to the **Actions** tab in your repo — you'll see the "Build and deploy to GitHub Pages" workflow running. Wait for the green checkmark (~1–2 minutes).
6. Back in **Settings → Pages**, refresh — you'll now see your live URL, something like:
   ```
   https://yourusername.github.io/nova-pwa/
   ```

### Alternative: plain git commands

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/yourusername/nova-pwa.git
git push -u origin main
```

Then do steps 4–6 above (enable Pages → Source: GitHub Actions) on GitHub.com.

### Updating the app later

Any time you change code and want it live again: commit and push (or in GitHub Desktop, type a commit message → **Commit** → **Push origin**). The Action rebuilds and redeploys automatically — no manual `npm run build` or file uploads needed.

## 4. Install it on an Android phone

1. Open the deployed HTTPS URL in Chrome on Android.
2. Tap the **⋮** menu → **Add to Home screen** (or wait for Chrome's automatic install banner).
3. It launches full-screen, no address bar, with the icon and dark status bar you configured in `vite.config.js`.

## 5. Where things live

| What | Where |
|---|---|
| App logic and UI | `src/App.jsx` (single component tree, no external state library) |
| Data persistence | `localStorage`, via the `usePersistentState` hook at the top of `App.jsx` — events, notes, tasks, and settings all survive reloads and reinstalls of the page (not of the app itself) |
| PWA manifest + service worker | Generated by `vite-plugin-pwa`, configured in `vite.config.js` |
| App icons | `public/icons/` — regenerate anytime by re-running the icon script below |
| Theme meta tags (status bar color, Apple home-screen support) | `index.html` |

## 6. Regenerating icons

The included icons echo the app's countdown-ring motif in emerald on zinc-950. To change the mark, edit and rerun this (requires Pillow: `pip install pillow`):

```python
from PIL import Image, ImageDraw

def make_icon(size, path, maskable=False):
    img = Image.new("RGBA", (size, size), (0, 0, 0, 0))
    d = ImageDraw.Draw(img)
    bg = (9, 9, 11, 255)
    accent = (52, 211, 153, 255)
    if maskable:
        d.rectangle([0, 0, size, size], fill=bg)
        pad = int(size * 0.22)
    else:
        d.rounded_rectangle([0, 0, size, size], radius=int(size * 0.22), fill=bg)
        pad = int(size * 0.24)
    cx, cy = size // 2, size // 2
    r = size // 2 - pad
    stroke = max(3, size // 22)
    d.ellipse([cx - r, cy - r, cx + r, cy + r], outline=(39, 39, 42, 255), width=stroke)
    d.arc([cx - r, cy - r, cx + r, cy + r], start=-90, end=180, fill=accent, width=stroke)
    rd = int(size * 0.07)
    d.ellipse([cx - rd, cy - rd, cx + rd, cy + rd], fill=accent)
    img.save(path)

make_icon(192, "public/icons/icon-192.png")
make_icon(512, "public/icons/icon-512.png")
make_icon(512, "public/icons/maskable-512.png", maskable=True)
make_icon(180, "public/icons/apple-touch-icon.png")
```

## 7. Notes on data

Everything (events, calendar notes, tasks, profile, accent color) is saved to the browser's `localStorage` on the device, scoped to the domain it's served from. There's no backend or sync — reinstalling from a different domain, clearing site data, or using a different browser starts fresh. The **Clear local storage** button in Settings wipes events, notes, and tasks (not your profile/theme settings) after a confirmation step.

To sync across devices later, swap `usePersistentState` for calls to a backend (e.g. Supabase, Firebase, or your own API) — the rest of the app already treats state as plain React state, so the swap is isolated to that one hook.
